## Por que um blog dentro do portfólio?

Manter um repositório editorial junto ao site facilita três coisas:

1. **Versionar conhecimento**: cada post vive no Git, com histórico, revisão por PR e CI/CD para deploy.
2. **Capitalizar projetos**: ao soltar um lab DevOps, escrevo aqui um post curto explicando arquitetura, decisões e armadilhas.
3. **Centralizar referências**: links do LinkedIn e do GitHub passam a apontar para um espaço próprio, soberano e sem algoritmos no caminho.

## Como adicionar um novo post

Crie um arquivo Markdown em `src/content/blog/` seguindo o padrão `YYYY-MM-DD-slug.md` e preencha o frontmatter:

```yaml
---
title: "Título da postagem"
description: "Resumo de 1 a 2 frases (importante para SEO e cards sociais)."
pubDate: 2026-06-01
tags: ["terraform", "aws"]
source: "github"               # artigo | github | linkedin
sourceUrl: "https://github.com/nilo-lima/repo"
draft: false                   # true esconde da listagem
---
```

Tudo o que vier depois é Markdown puro: headings, listas, blocos de código com syntax highlighting nativo do Astro, imagens em `public/blog/...`.
